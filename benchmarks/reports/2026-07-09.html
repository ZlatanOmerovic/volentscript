Reading the numbers honestly

  • Tight numeric loops (mandelbrot): within ~2x of C, at JS-JIT level — LLVM -O2 works when code stays in registers.
  • Call-heavy code (fib): ~6x C — a GC safepoint check per call, plus conservative-GC codegen constraints.
  • Object float math (nbody): ~2.8x C, down from ~10x — P29 inlined the per-field null check (was an opaque call that blocked load hoisting); 3.46x on this row in isolation.
  • Allocation churn (binarytrees): ~1.8x C, down from ~8x — P27 bump arenas replaced the per-object registry (3.68x on this binary); P28 parallelized the mark phase (~9% more, bandwidth-bound). Generational JIT collectors still win it outright.
  • Strings: ~8.6x C, down from ~13x and now ahead of Java — UTF-16-native ops (P26) plus the P27 allocation fast path; the churn from ~12 short-lived strings per iteration is the remaining gap.

Remaining gaps map to planned optimizations: a non-moving generational nursery for allocation churn, and module-level global storage so mutable top-level vars stop closure-converting their readers. None require language changes. Same idiomatic-simple algorithm in every language; Java timed as a process like everything else (its binarytrees throughput still wins). TypeScript is not a separate row — it erases to the same JS.

VolentScript — ActionScript 3, revived. Native, ahead-of-time, and entirely of its own will.
