Reading the numbers honestly

  • Tight numeric loops (mandelbrot): within ~2x of C, at JS-JIT level — LLVM -O2 works when code stays in registers.
  • Call-heavy code (fib): ~6x C — a GC safepoint check per call, plus conservative-GC codegen constraints.
  • Object float math (nbody): ~10x C — Vector.<T> element reads are runtime calls on boxed storage; safepoints block loop hoisting.
  • Allocation churn (binarytrees): ~8x C; generational JIT collectors win this workload outright.
  • Strings: ~12x C — every operation transcodes UTF-16 ↔ UTF-8 inside the runtime.

Each gap maps to a planned optimization: unboxed numeric Vectors, UTF-16-native string ops, allocation fast path + safepoint hoisting, generational GC. None require language changes. Same idiomatic-simple algorithm in every language; Java timed as a process like everything else (its binarytrees throughput still wins). TypeScript is not a separate row — it erases to the same JS.

VolentScript — ActionScript 3, revived. Native, ahead-of-time, and entirely of its own will.
